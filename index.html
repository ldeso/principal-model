Phase B — Fee vs. Principal Simulator

Interactive companion to the research note

Overview

This notebook is the Phase B companion to research-note.md. It reproduces the §2–§5 closed-form P&L moments, estimates the §4 tail-risk metrics by Monte Carlo, and lets the reader explore the (α, μ, σ) grid interactively. All Monte Carlo computation happens in TypeScript (src/); this page only reads the emitted JSON and visualises it.

Regenerate the data before previewing:

npm run simulate -- --seed 42
npm run sweep -- --seed 42

What you will see below. Four kinds of plots: (1) a scorecard table of closed-form vs Monte Carlo P&L moments with tail metrics, (2) overlaid P&L distributions for the three stochastic models, (3) diagnostics on the shared random kernel I_T := ∫₀ᵀ S_t dt, and (4) parameter sweeps showing how tail risk and the break-even quote Q* respond to drift μ, volatility σ, and hedge ratio α.

Notation cheat-sheet

A brief reminder of the symbols used throughout (full derivations in research-note.md §1–§5):

Symbol Meaning Units
S_t kVCM / USD spot price at time t USD / kVCM
P protocol price (constant) kVCM / tonne
π_t := P·S_t USD protocol price per tonne USD / tonne
λ deterministic retirement flow tonnes / year
T horizon years
N := λ·T total tonnes retired over [0,T] tonnes
f fee rate dimensionless
Q fixed USD quote per tonne (principal model) USD / tonne
α ∈ [0,1] pre-purchase (hedge) fraction dimensionless
I_T := ∫₀ᵀ S_t dt time-integral of GBM, the shared random kernel (USD/kVCM)·year
R_fee = f·P·λ·I_T fee-based revenue USD
Π_b2b = Q·N − P·λ·I_T back-to-back (α=0) P&L USD
Π_matched = N·(Q − P·S_0) matched pre-purchase (α=1), deterministic USD
Π_α = (1−α)·Π_b2b + α·Π_matched partial pre-purchase (α∈[0,1]) USD
Q* = (1+f)·P·S_0·(e^{μT}−1)/(μT) break-even quote equating E[Π_b2b] = E[R_fee] USD / tonne

§4 — P&L moments: closed-form vs Monte Carlo

Each row is one of the four models from the research note. The first block of columns cross-checks the Monte Carlo mean against the Dufresne-based closed form: ±CI95 is the ±95% half-width of the MC mean, and z is (MC − closed) / stderr. For large nPaths the MC mean should land within its own CI and |z| should rarely exceed 2. The second block of columns reports the risk quantities the closed form does not cover — empirical VaR, CVaR, loss probability, and the Sharpe-like ratio E[Π] / SD[Π]. All P&L numbers are USD over the horizon T.

column meaning
E[Π] (cf) / (mc) expected P&L — closed form vs Monte Carlo
±CI95 ±95% half-width of the MC mean
SD (cf) / (mc) standard deviation of P&L
z (MC−cf) / stderr; MC sanity check
VaR95 / VaR99 loss threshold exceeded with 5% / 1% probability
CVaR95 / CVaR99 expected loss in the worst 5% / 1% tail
P[Π<0] probability of a losing realisation
Sharpe E[Π] / SD[Π], invariant under f-scaling for fee model

principal_3a is deterministic at horizon, so SD, CI95, and z are zero by construction; VaR/CVaR simply reflect the sign of the matched P&L.

P&L distributions

Each panel is one stochastic model’s empirical density of terminal P&L, from a sub-sampled Monte Carlo cloud. The vertical black rule is the closed-form mean E[Π]; the dashed grey rule is Π = 0, so the mass to the left of it is the loss probability reported in the table.

  • Fee book (R_fee = f·P·λ·I_T): non-negative by construction, so mass sits entirely in the right half-plane. Distribution is right-skewed — a rescaled image of the I_T distribution.
  • Principal 3b (back-to-back, α = 0): linear decreasing in I_T, so the same kernel produces a left-skewed loss tail. Upside is capped at Q·N (reached as S_t → 0), downside is unbounded.
  • Principal 3c (partial, α = 0.5 in this run): variance scales as (1−α)² relative to 3b, so the density is narrower.

I_T histogram vs. Dufresne closed-form moments

I_T := ∫₀ᵀ S_t dt is the shared random kernel driving both the fee book and the principal back-to-back book. Its first two moments are known in closed form (Dufresne, 2001), but the distribution itself is not log-normal, so tail quantities must be estimated numerically. This plot is a Monte Carlo histogram of I_T — the solid rule verifies that the empirical mean matches Dufresne’s E[I_T] = S_0·(e^{μT}−1)/μ; the dashed rules sit at E[I_T] ± SD.

itHist = {
  const { edges, counts } = run.itSampleHistogram;
  const total = counts.reduce((a,b) => a+b, 0);
  return counts.map((c,i) => ({
    x0: edges[i], x1: edges[i+1], density: c / (total * (edges[i+1]-edges[i]))
  }));
}
Plot.plot({

  height: 320,

  y: { label: "empirical density", labelAnchor: "center" },
  marks: [
    Plot.rectY(itHist, { x1: "x0", x2: "x1", y: "density", fill: "#4c78a8", fillOpacity: 0.7 }),

Sample GBM paths — matched-book NAV view (§3a)

The Phase A note defines the mark-to-market inventory value for the fully matched pre-purchase book (α = 1) as

V_t = (N · P) · (1 − t/T) · S_t.

This is a scaled GBM attenuated by the linear burn schedule. V_0 = N·P·S_0 is the purchase cost; V_T = 0 because inventory has been fully retired by the horizon. The plot shows a handful of sampled paths of V_t — the highlighted orange line is the first path, the grey lines are the rest. Interim drawdowns in V_t are the solvency concern even though terminal P&L is deterministic.

pathRecords = run.sampledPaths.flatMap((path, idx) =>
  path.map((s, k) => ({
    t: k / run.params.nSteps,
    S: s,
    V: run.params.lambda * run.params.T * run.params.P * (1 - k / run.params.nSteps) * s,
    id: idx
  }))
)

NAV shortfall vs. schedule (§3a)

For the matched book, a meaningful solvency statistic is the worst shortfall against the expected decay schedule:

D := max_{t ≤ T}  [ N·P·(1 − t/T) · (S_0 − S_t) ]₊

i.e., the deepest excursion of V_t below its pre-hedged deterministic path. This histogram is the empirical distribution of D across paths. Values on the x-axis are USD losses relative to schedule.

Break-even quote surface Q*(μ, T)

Q* is the USD quote that equalises the expected P&L of the back-to-back principal book with the fee-based book:

Q* = (1 + f) · P · S_0 · (e^{μT} − 1) / (μ T).

Normalising by (1+f)·P·S_0 removes the trivial scaling and leaves only the drift-horizon shape (e^{μT}−1)/(μT). The dashed horizontal rule is the μ = 0 limit: at zero drift, Q* = (1+f)·P·S_0, i.e. the fee-based time-zero quote.

Reading notes

  • All closed-form / Monte Carlo agreement checks live in the z column of the §4 table — a single glance should tell you whether the MC is well-converged at the current nPaths.
  • The fee and back-to-back books share the same random kernel I_T, so variance ratios are exact algebraic multiples ((P/f)²) — deviations would indicate a simulator bug.
  • This notebook is Phase B; Phase C extensions (jump diffusion, Poisson demand, historical calibration, dynamic hedging) are intentionally out of scope here.
